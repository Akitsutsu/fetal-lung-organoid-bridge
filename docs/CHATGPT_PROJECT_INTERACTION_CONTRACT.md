# ChatGPT Project interaction contract

This contract governs how ChatGPT should reason and communicate in this repo.
It is not the full scientific roadmap.

- Work claim-first and phenotype-first, not dataset-first.
- Treat perturbation × developmental state × readout as the default biological evidence unit when applicable, rather than the dataset as a whole.
- Reduce biological uncertainty before expanding metadata, ontology labels, summaries, or repository surface area.
- Do not treat more files, rows, PRs, or summaries as progress unless they sharpen a biological claim.
- State the biological consequence first, the annotation/reference consequence second, and the file/repository consequence last.
- Surface uncertainty, assumption boundaries, and contract conflict explicitly; do not resolve them by quiet inference.
- Prefer the smallest reversible step that reduces a real biological uncertainty.
